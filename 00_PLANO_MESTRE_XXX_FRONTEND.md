> Objetivo final: construir o eixo vertical do seu T como **Front-End Software Engineer** com perfil compatível com vagas como **Software Engineer (Front End), Retail and Marcom Engineering** da XXX.

---

## 1. Seu ponto de partida

Você já tem a base horizontal do T:

- back-end;
- APIs;
- CI/CD;
- containers;
- deploy;
- noção de arquitetura;
- experiência full-stack.

Agora o foco não é virar “full-stack genérico”. O foco é criar profundidade real em front-end.

O objetivo é sair de:

> Desenvolvedor full-stack que também faz telas.

Para:

> Front-End Software Engineer especializado em JavaScript, React, Design Systems, CSS em escala, acessibilidade, performance e arquitetura de UI, com base full-stack suficiente para conversar com back-end, DevOps e infraestrutura.

---

## 2. O que a vaga real da XXX está pedindo

A vaga que você trouxe aponta para um perfil muito específico:

- construir e manter **shared UI components**;
- trabalhar em uma **component library**;
- usar HTML, CSS e JavaScript;
- atuar com React durante uma migração de arquitetura;
- trabalhar com Node.js tooling;
- usar NPM, webpack, Vite ou similares;
- pensar em component APIs, variants e interaction patterns;
- contribuir em decisões arquiteturais;
- dominar CSS architecture;
- conhecer BEM, CSS Modules, Styled Components ou similares;
- otimizar performance;
- aplicar accessibility/WCAG;
- fazer cross-browser testing;
- documentar componentes com Storybook ou equivalente;
- comunicar bem com times técnicos, criativos e de design.

Portanto, seu estudo precisa ser orientado para **UI Engineering em escala**, não apenas para “aprender React”.
