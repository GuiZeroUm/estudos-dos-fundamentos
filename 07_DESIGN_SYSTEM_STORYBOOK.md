# Design System e Storybook

> Objetivo: construir o tipo de experiência diretamente alinhada à vaga da XXX: component library, APIs de componentes, variantes, documentação, acessibilidade e colaboração com design.

---

## 1. O que dominar

- component library;
- design tokens;
- variants;
- states;
- Storybook;
- docs;
- playground de props;
- versionamento;
- changelog;
- testes;
- acessibilidade;
- integração com Figma/design;
- component API design.

---

## 2. Projeto principal — React Component Library

Stack sugerida:

- React;
- TypeScript;
- Vite;
- Storybook;
- CSS Modules ou Styled Components;
- Vitest;
- React Testing Library;
- Playwright;
- GitHub Actions.

- Button;

- Tabs;

- Tooltip;

---

## 4. Critérios para cada componente

Cada componente deve ter:

- API clara;
- variantes;
- estados;
- acessibilidade;
- keyboard navigation quando necessário;
- testes;
- documentação no Storybook;
- exemplos reais;
- design tokens;
- controle de foco;
- suporte a tema.

---

## 5. Exemplo de documentação por componente

Para cada componente, documente:

- propósito;
- quando usar;
- quando não usar;
- props;
- exemplos;
- estados;
- acessibilidade;
- comportamento por teclado;
- decisões técnicas.
