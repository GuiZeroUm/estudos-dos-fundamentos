## 2. Como escrever README técnico

Cada README deve ter:

- objetivo;
- stack;
- arquitetura;
- decisões técnicas;
- trade-offs;
- como rodar;
- como testar;
- acessibilidade;
- performance;

## 3. Frases fortes para currículo

Use quando forem verdadeiras:

- Built reusable React components with documented APIs and accessibility support.
- Developed a scalable CSS architecture using design tokens and component-scoped styling.
- Created a Storybook-based component library with tests and usage documentation.
- Improved front-end performance through bundle optimization and render profiling.
- Implemented accessible UI primitives including modal, tabs, accordion and tooltip.
- Migrated vanilla JavaScript UI patterns into reusable React components.
- Configured CI pipelines for linting, tests, build and deployment.

## 5. O que evitar

- CRUD genérico sem acabamento.
- Projeto sem README.
- Código sem testes.
- Interface sem acessibilidade.
- CSS bagunçado.
- React sem arquitetura.
- Projeto que parece tutorial copiado.
- Portfólio apenas visual, sem engenharia.
